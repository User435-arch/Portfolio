Réaliser un développement d'application

Cette compétence consiste à développer c'est-à-dire concevoir, coder, tester et intégrer une solution informatique pour un client. elle repose sur 4 Apprentissages critiques :

Implémenter des conceptions simples

Dans cet exemple, j'ai du réaliser un programme Java visant à simuler un diaporama, pour cela, j'ai du créer plusieurs classes telles que diapositives, diaporama pour placer des diapositives. ainsi que les titres, textes et position. Ensuite j'ai regroupé ces éléments avec une relation d'héritage autour d'une classe mère élément (généralisation) pour rendre le code modulaire.

Elaborer des conceptions simples

Dans cet exemple, j'ai réalisé un diagramme de classes pour la SAE 2.01 qui a consisté à développer une application java de gestion de graphes MPM qui affiche les différentes tâches à partir d'un fichier texte. Ce diagramme était l'objet d'un prototype simple permettant constituer la base de ce projet avant de l'enrichir.

Veiller à la qualité du code

Dans cet exemple, j'ai développé un jeu de plateforme en Java. Nous pouvons observer que j'ai utilisé des noms de variables évocateurs. J'ai découpé le code en méthodes pour faciliter les modifications. J'ai également écrit des commentaires qui constituent une documentation, ce qui facilite la réutilisation du code.

Faire des essais et évaluer leurs résultats au regard de leurs spécifications

Dans ce projet, lors duquel j'ai codé un calendrier grégorien en java, j'ai développé une méthode qui détermine si l'année passée en paramètre est bissextile et une autre pour caluler le nombre de jours dans un mois. J'ai ensuite vérifié le fonctionnement de ces méthodes par des tests comme pour les mois de février 2000 ou avril 1900 afin de m'assurer qu'elles respectent bien les spécifications du calendrier grégorien.

Le code du calendrier grégorien

Développer des interfaces utilisateur en respectant des normes

Dans cet exemple, j'ai développé une interface graphique Java qui consiste en une fenêtre contenant une barre de menu. L’interaction avec ces menus affiche l'action associée dans la console. Exemple : pour l'entrée du menu ouvrir, on affiche dans la console le mot "ouvrir". Afin de respecter les principes d'ergonomie, j'ai configuré des raccourcis clavier pour chacun de ces menus tels que "ctrl + S" pour sauvegarder ou "ctrl + O" pour ouvrir. La barre du menu à été codée dans une classe dédiée pour respecter la modularité et rendre le code plus accessible.
